## 1. 产品概述

### 1.1 产品背景

AI 编程工具的快速发展大幅降低了 iOS App 开发门槛，但上架流程依然复杂：
- Apple Developer 账号申请流程繁琐
- 证书/配置文件管理混乱
- App Store Connect 元数据填写复杂（多语言、多尺寸截图）
- 年龄分级、隐私标签等专业性强
- 审核被拒后不知如何应对

大量独立开发者和小型团队在"最后一公里"卡住，需要一个平台帮助完成整个上架流程。

### 1.2 产品定位

**iOS Kit** 是一款面向独立开发者和小型团队的 iOS App 上架辅助平台，通过 AI 能力和自动化工具，让上架 iOS App 像填空一样简单。

### 1.3 目标用户

| 用户类型 | 特征 | 核心需求 |
|---------|------|---------|
| 独立开发者 | 技术背景较弱，想用 AI 快速做 App 变现 | 全流程指引、减少手动操作 |
| 小型团队 | 有基本开发能力，缺乏 iOS 上架经验 | 合规检查、避免审核被拒 |
| 工作室/企业 | 批量管理多个 App | 效率工具、权限管理 |

### 1.4 核心价值主张

- **简单**：像填空一样完成上架配置
- **智能**：AI 生成元数据、自动检查合规
- **省钱**：避免反复被拒产生的时间成本
- **可信赖**：透明安全，不触碰用户敏感权限

## 2. 用户画像与场景

### 2.1 典型用户：小张（独立开发者）

**背景**：
- 28 岁，前端开发，业余做副业
- 用 AI 工具开发了一款效率 App
- 第一次尝试上架 iOS，被 Apple 各种配置搞晕

**使用场景**：
1. 注册 Apple Developer 账号，看平台指引视频
2. 用平台生成隐私政策、用户协议
3. 用 AI 生成 App 描述、关键词（中英双语）
4. 上传原始截图，平台自动生成全尺寸
5. 完成年龄分级问卷，生成填写建议
6. 对照 Checklist 提审，一次通过

**痛点**：
- 不知道证书怎么配置
- 截图要多个尺寸，手动搞太麻烦
- 隐私标签不知道咋填
- 怕被拒，不知道哪里会出问题

### 2.2 典型用户：Lisa（小型团队创始人）

**背景**：
- 35 岁，创业做教育类 App
- 团队 5 人，有开发但无 iOS 经验
- 需要尽快上架验证商业模式

**使用场景**：
1. 团队多人协作管理项目进度
2. 用合规预审功能提前检查风险
3. AI 生成多语言元数据（中/英/日）
4. 批量生成 iPad 和 iPhone 全尺寸截图
5. 对接 App Store Connect API 自动提交

**痛点**：
- 时间紧，需要快速上线
- 多语言本地化工作量大
- 担心合规问题导致延期

## 4. 技术架构

### 4.1 系统架构图

```
┌─────────────────────────────────────────────────────────┐
│              前端 (React + Vite)                        │
│  · 项目管理界面  · 截图编辑器  · 🤖 AI 助手悬浮窗        │
└─────────────────────────────────────────────────────────┘
                            ↕
┌─────────────────────────────────────────────────────────┐
│              后端 (Python FastAPI)                      │
│  · AI 助手逻辑  · 图像处理  · 隐私文档生成  │
└─────────────────────────────────────────────────────────┘
                            ↕
┌─────────────────────────────────────────────────────────┐
│                  Supabase                               │
│  · Auth (用户认证)  · Postgres (数据)  · Storage (文件) │
└─────────────────────────────────────────────────────────┘
```

### 4.2 技术栈选型

| 层级 | 技术 | 选择理由 |
|------|------|---------|
| 前端 | React 18 + Vite | 轻量快速、生态丰富 |
| UI | Tailwind CSS + shadcn/ui | 快速开发、美观 |
| 后端 | Python FastAPI | AI 生态好、开发效率高 |
| BaaS | Supabase | 用户认证 + 数据库 + 存储一站式 |
| AI | Claude API | 文案生成质量高 |
| 截图渲染 | Playwright + Pillow | HTML 渲染 + 图像处理 |
| 部署 | Vercel (前端) + Railway (后端) | 快速上线 |

### 4.3 Supabase 方案

```
┌─────────────────────────────────────────────────────┐
│                  Supabase                           │
├─────────────────────────────────────────────────────┤
│ 1. Auth - 用户认证（邮箱/Google/Apple）             │
│ 2. Postgres - 数据库（用户数据/App 数据）            │
│ 3. Storage - 文件存储（截图/文档）                   │
│ 4. Edge Functions - 服务端逻辑（可选）               │
└─────────────────────────────────────────────────────┘
```

**优势**：
- 用户系统开箱即用，不用自己实现
- 数据库 + 存储一体化
- 免费额度够 MVP 使用
- 后续可扩展自托管

**HTML 模板示例**：

```html
<!-- iPhone 设备框模板 -->
<div class="screenshot" style="width: 1284px; height: 2778px;">
  <!-- 模糊背景 -->
  <div class="background" style="
    background: linear-gradient(135deg, #667eea 0%, #764ba2 100%);
    backdrop-filter: blur(20px);
  "></div>

  <!-- 设备边框 -->
  <div class="device-frame" style="
    width: 1200px;
    height: 2600px;
    border: 4px solid #333;
    border-radius: 50px;
    overflow: hidden;
  ">
    <!-- 用户截图 -->
    <img src="user-upload.png" style="width: 100%; height: 100%; object-fit: cover;" />
  </div>

  <!-- 可选：文案/Logo -->
  <div class="caption">Fitness Pro - 你的私人健身教练</div>
```

**多尺寸配置**：

| 设备 | 分辨率 | CSS 尺寸 |
|------|--------|---------|
| iPhone 6.5" | 1284x2778 | 414x896 @3x |
| iPhone 6.7" | 1290x2796 | 430x932 @3x |
| iPad 12.9" | 2048x2732 | 1024x1366 @2x |
| iPad 10.9" | 1640x2360 | 820x1180 @2x |

**优势**：
- 模板用 HTML/CSS 编写，设计师可轻松修改
- 支持任意 Web 样式（渐变、阴影、动画）
- 可添加文案、Logo、装饰元素
- 一套模板输出全部尺寸

## 5. 交互设计

### 5.1 核心流程

```
用户注册 → 创建项目 → 填写 App 信息 → AI 生成元数据
    ↓
上传截图 → 美化/多尺寸生成 → 完成合规检查
    ↓
导出全部材料 → 手动/API 提交 App Store Connect
```

### 5.2 🤖 AI 助手设计（核心卖点）

**入口**：右侧固定悬浮按钮，点击展开 AI 面板

**布局**：双栏 + 右侧 AI 面板

```
┌─────────────┬─────────────────────────┬──────────────┐
│  侧边导航   │     主内容区            │   AI 助手    │
│  (固定)     │     (自适应)            │   (可收起)   │
│             │                         │              │
│  项目列表   │  当前编辑内容           │  🤖 对话     │
│  + 新建     │  名称/描述/关键词       │  生成结果    │
│             │  [✨ AI 生成]           │  一键填充    │
└─────────────┴─────────────────────────┴──────────────┘
```

**AI 面板状态**：
- **收起**：仅显示右侧悬浮按钮
- **展开**：380px 宽度面板，显示对话
- **生成中**：显示进度条和步骤

**交互模式**：
```
┌─────────────────────────────────────────┐
│  🤖 iOS 助手                      [─] [×]│
├─────────────────────────────────────────┤
│                                         │
│  👋 你好！我可以帮你：                   │
│  • 生成 App 名称、描述、关键词            │
│  • 处理截图，生成全尺寸                 │
│  • 创建隐私政策和用户协议               │
│                                         │
│  ───────────────────────────────────── │
│                                         │
│  💬 试一试对我说：                       │
│  "帮我为这个 App 生成上架材料"             │
│  "把描述改得更吸引人"                   │
│  "生成隐私政策"                         │
│                                         │
│  ───────────────────────────────────── │
│                                         │
│  [💬 输入消息...                  ] [➤]│
│                                         │
└─────────────────────────────────────────┘
```

**AI 助手能力**：

| 意图 | 用户指令示例 | AI 响应 |
|------|------------|--------|
| 生成元数据 | "帮我想几个 App 名称" | 生成 5 个名称 + 副标题建议 |
| 生成描述 | "写一个 200 字的介绍" | 生成 App Store 描述文案 |
| 生成关键词 | "给我一些关键词" | 生成 100 字符内关键词列表 |
| 处理截图 | "把截图做成全尺寸" | 引导上传 → 调用截图工厂 |
| 生成隐私 | "我需要隐私政策" | 问卷式询问 → 生成文档 |
| 修改调整 | "把描述改得更正式" | 基于原文案重新生成 |
| 一键全包 | "帮我生成全部上架材料" | 依次触发元数据 + 截图 + 隐私 |

**上下文感知**：
- AI 助手知道当前选中的是哪个 App 项目
- 可以读取已保存的 App 信息作为生成上下文
- 生成的内容自动保存到对应项目

### 5.3 关键界面

#### 首页 / Dashboard
- 项目列表（卡片式）
- 快速创建入口
- AI 助手悬浮按钮（右下角）

#### 项目详情页
- Tab 1: 基本信息（Bundle ID、分类等）
- Tab 2: AI 元数据（名称、描述、关键词）
- Tab 3: 截图工厂
- Tab 4: 隐私文档
- 侧边或底部：AI 助手悬浮窗
