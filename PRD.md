## 1. 产品概述

### 1.1 产品背景

AI 编程工具的快速发展大幅降低了 iOS App 开发门槛，但上架流程依然复杂：
- Apple Developer 账号申请流程繁琐
- 证书/配置文件管理混乱
- App Store Connect 元数据填写复杂（多语言、多尺寸截图）
- 年龄分级、隐私标签等专业性强
- 审核被拒后不知如何应对

大量独立开发者和小型团队在"最后一公里"卡住，需要一个平台帮助完成整个上架流程。

### 1.2 产品定位

**iOS Kit** 是一款面向独立开发者和小型团队的 iOS App 上架辅助平台，通过 AI 能力和自动化工具，让上架 iOS App 像填空一样简单。

### 1.3 目标用户

| 用户类型 | 特征 | 核心需求 |
|---------|------|---------|
| 独立开发者 | 技术背景较弱，想用 AI 快速做 App 变现 | 全流程指引、减少手动操作 |
| 小型团队 | 有基本开发能力，缺乏 iOS 上架经验 | 合规检查、避免审核被拒 |
| 工作室/企业 | 批量管理多个 App | 效率工具、权限管理 |

### 1.4 核心价值主张

- **简单**：像填空一样完成上架配置
- **智能**：AI 生成元数据、自动检查合规
- **省钱**：避免反复被拒产生的时间成本
- **可信赖**：透明安全，不触碰用户敏感权限

## 2. 用户画像与场景

### 2.1 典型用户：小张（独立开发者）

**背景**：
- 28 岁，前端开发，业余做副业
- 用 AI 工具开发了一款效率 App
- 第一次尝试上架 iOS，被 Apple 各种配置搞晕

**使用场景**：
1. 注册 Apple Developer 账号，看平台指引视频
2. 用平台生成隐私政策、用户协议
3. 用 AI 生成 App 描述、关键词（中英双语）
4. 上传原始截图，平台分析页面内容并生成可编辑截图方案
5. 在前端画布中拖拽微调标题、截图、Logo 和装饰元素
6. 完成年龄分级问卷，生成填写建议
7. 对照 Checklist 提审，一次通过

**痛点**：
- 不知道证书怎么配置
- 截图要多个尺寸，手动搞太麻烦
- 隐私标签不知道咋填
- 怕被拒，不知道哪里会出问题

### 2.2 典型用户：Lisa（小型团队创始人）

**背景**：
- 35 岁，创业做教育类 App
- 团队 5 人，有开发但无 iOS 经验
- 需要尽快上架验证商业模式

**使用场景**：
1. 团队多人协作管理项目进度
2. 用合规预审功能提前检查风险
3. AI 生成多语言元数据（中/英/日）
4. 批量生成 iPad 和 iPhone 审核/商店截图，并支持人工微调
5. 对接 App Store Connect API 自动提交

**痛点**：
- 时间紧，需要快速上线
- 多语言本地化工作量大
- 担心合规问题导致延期

## 4. 技术架构

### 4.1 系统架构图

```
┌─────────────────────────────────────────────────────────┐
│              前端 (React + Vite)                        │
│  · 项目管理界面  · 截图编辑器  · 🤖 AI 助手悬浮窗        │
└─────────────────────────────────────────────────────────┘
                            ↕
┌─────────────────────────────────────────────────────────┐
│              后端 (Python FastAPI)                      │
│  · AI 助手逻辑  · 设计状态编排  · 导出授权  · 隐私文档生成 │
└─────────────────────────────────────────────────────────┘
                            ↕
┌─────────────────────────────────────────────────────────┐
│                  Supabase                               │
│  · Auth (用户认证)  · Postgres (数据)  · Storage (文件) │
└─────────────────────────────────────────────────────────┘
```

### 4.2 技术栈选型

| 层级 | 技术 | 选择理由 |
|------|------|---------|
| 前端 | React 18 + Vite | 轻量快速、生态丰富 |
| UI | Tailwind CSS + shadcn/ui | 快速开发、美观 |
| 后端 | Python FastAPI | AI 生态好、开发效率高 |
| BaaS | Supabase | 用户认证 + 数据库 + 存储一站式 |
| AI | Claude API | 文案生成质量高 |
| 截图编辑 | React + Konva | 前端画布编辑、拖拽微调、实时预览 |
| 导出 | Canvas Export + ZIP | 同一份 Scene Spec 输出预览图和下载包 |
| 部署 | Vercel (前端) + Railway (后端) | 快速上线 |

### 4.3 Supabase 方案

```
┌─────────────────────────────────────────────────────┐
│                  Supabase                           │
├─────────────────────────────────────────────────────┤
│ 1. Auth - 用户认证（邮箱/Google/Apple）             │
│ 2. Postgres - 数据库（用户数据/App 数据）            │
│ 3. Storage - 文件存储（截图/文档）                   │
│ 4. Edge Functions - 服务端逻辑（可选）               │
└─────────────────────────────────────────────────────┘
```

**优势**：
- 用户系统开箱即用，不用自己实现
- 数据库 + 存储一体化
- 免费额度够 MVP 使用
- 后续可扩展自托管

### 4.4 截图工厂 - 前端可编辑画布架构

```
┌─────────────────────────────────────────────────────────┐
│                    截图工厂流程                         │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  用户创建项目（名称/描述/Logo）                         │
│       ↓                                                 │
│  用户上传截图（多张）                                   │
│       ↓                                                 │
│  ┌─────────────────────────────────────────────────┐   │
│  │             AI 规划层                            │   │
│  │  · 理解截图内容 / OCR                            │   │
│  │  · 生成 Brand Kit                                │   │
│  │  · 生成 Storyboard                               │   │
│  │  · 生成 Scene Spec(JSON)                         │   │
│  └─────────────────────────────────────────────────┘   │
│       ↓                                                 │
│  ┌─────────────────────────────────────────────────┐   │
│  │        前端 Screenshot Editor（画布）           │   │
│  │  · React + Konva 实时渲染                       │   │
│  │  · 拖拽/缩放/对齐线/图层调整                     │   │
│  │  · 自然语言修改后即时重绘                         │   │
│  └─────────────────────────────────────────────────┘   │
│       ↓                                                 │
│  低清预览（免费） / 高清导出（付费）                    │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

**核心设计原则**：
- AI 只负责生成结构化设计状态，不直接产出最终 HTML。
- 前端画布是唯一预览真相来源，拖拽调整和自然语言调整都回写同一份 `Scene Spec`。
- 免费用户可实时预览；付费后解锁高清导出与批量下载。
- MVP 不依赖 Playwright，避免后端渲染链路带来的并发和部署复杂度。

**Scene Spec 示例**：

```json
{
  "page_id": "page_01",
  "device_type": "iphone_67",
  "brand_kit": {
    "theme": "clean_fitness",
    "palette": ["#0B1220", "#EAF2FF", "#31C48D"],
    "heading_font": "Space Grotesk",
    "body_font": "Inter",
    "logo_mode": "top_left"
  },
  "elements": [
    {
      "id": "headline_1",
      "type": "text",
      "content": "训练计划，一眼就能开始",
      "x": 84,
      "y": 120,
      "width": 920,
      "font_size": 84,
      "font_weight": 700
    },
    {
      "id": "screen_1",
      "type": "image",
      "source": "upload://shot_01",
      "x": 126,
      "y": 420,
      "width": 1038,
      "height": 2120,
      "corner_radius": 56
    }
  ]
}
```

**多尺寸配置**：

| 设备 | 分辨率 | CSS 尺寸 |
|------|--------|---------|
| iPhone 6.5" | 1284x2778 | 414x896 @3x |
| iPhone 6.7" | 1290x2796 | 430x932 @3x |
| iPad 12.9" | 2048x2732 | 1024x1366 @2x |
| iPad 10.9" | 1640x2360 | 820x1180 @2x |

**核心数据模型**：
- `Brand Kit`：主题风格、配色、字体、Logo 使用规则、设备框策略。
- `Storyboard`：截图顺序、每页卖点、主副标题、模板意图。
- `Scene Spec`：可直接驱动画布渲染的结构化 JSON，记录元素类型、位置、尺寸、样式和层级。

**优势**：
- 预览和编辑都发生在前端，反馈即时，适合所见即所得。
- 自然语言调整和手动拖拽共享一份状态，不会出现“预览一套、导出一套”。
- 设计系统扩展性更强，后续可接模板市场、团队协作和版本历史。
- 避免浏览器自动化渲染链路，降低部署和并发复杂度。

## 5. 交互设计

### 5.1 核心流程

```
用户注册 → 创建项目 → 填写名称/描述/上传 Logo
    ↓
上传截图 → AI 生成 Brand Kit / Storyboard / Scene Spec
    ↓
前端画布预览与拖拽微调 / 自然语言改 UI
    ↓
付费解锁高清导出 → 打包全部材料 → 手动/API 提交 App Store Connect
```

### 5.2 🤖 AI 助手设计（核心卖点）

**入口**：右侧固定悬浮按钮，点击展开 AI 面板

**布局**：双栏 + 右侧 AI 面板

```
┌─────────────┬─────────────────────────┬──────────────┐
│  侧边导航   │     主内容区            │   AI 助手    │
│  (固定)     │     (自适应)            │   (可收起)   │
│             │                         │              │
│  项目列表   │  当前编辑内容           │  🤖 对话     │
│  + 新建     │  名称/描述/关键词       │  生成结果    │
│             │  [✨ AI 生成]           │  一键填充    │
└─────────────┴─────────────────────────┴──────────────┘
```

**AI 面板状态**：
- **收起**：仅显示右侧悬浮按钮
- **展开**：380px 宽度面板，显示对话
- **生成中**：显示进度条和步骤

**交互模式**：
```
┌─────────────────────────────────────────┐
│  🤖 iOS 助手                      [─] [×]│
├─────────────────────────────────────────┤
│                                         │
│  👋 你好！我可以帮你：                   │
│  • 生成 App 名称、描述、关键词            │
│  • 分析截图，生成可编辑审核图            │
│  • 创建隐私政策和用户协议               │
│                                         │
│  ───────────────────────────────────── │
│                                         │
│  💬 试一试对我说：                       │
│  "帮我为这个 App 生成上架材料"             │
│  "把描述改得更吸引人"                   │
│  "生成隐私政策"                         │
│                                         │
│  ───────────────────────────────────── │
│                                         │
│  [💬 输入消息...                  ] [➤]│
│                                         │
└─────────────────────────────────────────┘
```

**AI 助手能力**：

| 意图 | 用户指令示例 | AI 响应 |
|------|------------|--------|
| 生成元数据 | "帮我想几个 App 名称" | 生成 5 个名称 + 副标题建议 |
| 生成描述 | "写一个 200 字的介绍" | 生成 App Store 描述文案 |
| 生成关键词 | "给我一些关键词" | 生成 100 字符内关键词列表 |
| 处理截图 | "把截图做成 App Store 审核图" | 引导上传 → 生成 Brand Kit / Storyboard / Scene Spec |
| 调整 UI | "第二张标题大一点，Logo 放左上" | 修改 Scene Spec → 画布即时重绘 |
| 生成隐私 | "我需要隐私政策" | 问卷式询问 → 生成文档 |
| 修改调整 | "把描述改得更正式" | 基于原文案重新生成 |
| 一键全包 | "帮我生成全部上架材料" | 依次触发元数据 + 截图 + 隐私 |

**上下文感知**：
- AI 助手知道当前选中的是哪个 App 项目
- 可以读取已保存的 App 信息作为生成上下文
- 生成的内容自动保存到对应项目
